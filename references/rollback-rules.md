# Rollback Rules

Use this when a problem discovered in review actually started upstream in the workflow.

## Principle

Do not repair upstream analysis errors only by rewriting the final report.

## Common rollback paths

### Linkage or matching problem

Roll back to:
- mixed-method linkage step
- quantitative cleaning if needed
- qualitative cleaning if needed

### Wrong subgrouping or weak quantitative cuts

Roll back to:
- quantitative segmentation and cross-tab stage

### Weak qualitative pattern or inconsistent coding

Roll back to:
- qualitative immersion
- qualitative coding
- case comparison

### Unsupported integrated claim

Roll back to:
- mixed-method integration stage

### Insight too shallow or overclaimed

Roll back to:
- insight formation after reviewing single-source and integration evidence

### Report missing charts, tables, or traceability

Roll back to:
- the phase that should have produced the artifact, not only report writing

## Rule of thumb

If a correction changes the underlying evidence, go upstream.
If a correction changes only framing or readability, local report revision is enough.
